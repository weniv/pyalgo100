# 디스코드 채팅방
## 문제 설명
위니브에서는 실제 디스코드로 [위니빌리지](https://discord.gg/MASRTmyu8a)라는 ICT 교육을 배우는 사람의 모임을 운영하고 있습니다. 디스코드에서는 멤버가 서버에 입장하면 "OOO님이 착륙하셨어요."라는 시스템 메시지가 표시됩니다.

디스코드는 유저마다 고유한 ID가 있고, 닉네임은 자유롭게 변경할 수 있습니다. 닉네임이 변경되면 이전에 표시된 시스템 메시지의 닉네임도 새 닉네임으로 업데이트됩니다. 멤버가 서버를 나갈 때는 별도의 시스템 메시지가 표시되지 않습니다.

예를 들어, 나중에 들어온 사람이 먼저 들어온 사람이 들어온 입장 기록을 보면, 항상 그 기록은 최신의 닉네임으로 표시됩니다.

서버의 활동 로그가 담긴 문자열 배열 `logs`가 주어질 때, 모든 로그가 처리된 후 최종적으로 표시될 시스템 메시지를 문자열 배열로 return 하도록 solution 함수를 완성해주세요.

- `In`: 멤버가 서버에 입장
- `Update`: 멤버가 닉네임을 변경

※ 퇴장 시에는 로그가 남지 않습니다.

---

## 제한 사항

- `logs`의 길이는 1 이상 100 이하입니다.
- 각 문자열은 공백으로 구분되어 있으며, 다음 형식을 따릅니다.
  - `In [유저ID] [닉네임]`
  - `Update [유저ID] [닉네임]`
- 유저ID와 닉네임은 알파벳 소문자와 숫자로만 이루어져 있습니다.
- 유저ID와 닉네임의 길이는 1 이상 10 이하입니다.
- 같은 유저가 여러 번 입장할 수 있으며, 재입장 시 새로운 닉네임으로 입장할 수 있습니다.

---

## 입출력 예

| logs | result |
| ---- | ------ |
| ["In user1 Licat", "In user2 Mura", "In user1 Mura", "Update user2 Binky"] | ["Mura님이 착륙하셨어요.", "Binky님이 착륙하셨어요.", "Mura님이 착륙하셨어요."] |
| ["In user1 Alex", "Update user1 Bob", "In user1 Charlie"] | ["Charlie님이 착륙하셨어요.", "Charlie님이 착륙하셨어요."] |
| ["In user1 A", "In user2 B", "Update user1 C", "In user2 D"] | ["C님이 착륙하셨어요.", "D님이 착륙하셨어요.", "D님이 착륙하셨어요."] |

---

## 입출력 설명
- 첫 번째 예시:
  1. user1이 "Licat"으로 입장 → 나중에 "Mura"로 재입장하여 닉네임 변경됨
  2. user2가 "Mura"로 입장 → 나중에 "Binky"로 닉네임 변경됨
  3. user1이 "Mura"로 재입장
  4. 최종: user1은 "Mura", user2는 "Binky"

- 두 번째 예시:
  - user1이 "Alex"로 입장 → "Bob"으로 닉네임 변경 → "Charlie"로 재입장
  - 최종 닉네임 "Charlie"로 모든 메시지가 표시됨

- 세 번째 예시:
  - user1: A로 입장 → C로 닉네임 변경
  - user2: B로 입장 → D로 재입장
  - 최종: user1은 "C", user2는 "D"
